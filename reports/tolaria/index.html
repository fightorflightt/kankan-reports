#知识库 #Markdown 笔记 #本地优先 #AI 友好

这是什么？

一句话:一个管理 Markdown 笔记的桌面 App,类似 Obsidian/Notion 的"第二大脑"工具。支持 Mac/Windows/Linux。你的所有笔记都是普通的 .md 文本文件,存在你自己电脑上,不锁在任何平台里。

它最核心的三个主张:文件优先(笔记就是纯文本,随时能用别的工具打开、想走就走)、本地优先(不用注册、不用订阅、不联网也能用)、AI 友好(因为是一堆纯文本文件,AI 助手能直接读写,可以拿来当"喂给 AI 的公司知识库"或"AI 助手的记忆")。作者是知名工程类 Newsletter「Refactoring」的主理人 Luca Rossi,他自己用它管着 1 万多条笔记、"用它来运营整个人生"。

应用场景

个人第二大脑：把日记、灵感、读书笔记都存成本地 md 文件，永不被某个平台绑架，想换工具直接带走
给 AI 配知识库：把公司文档/资料整理成一个 vault，让 Claude Code 等 AI 助手随时读取、问答、整理
AI 助手的长期记忆：存放 AI 助手的记忆和操作流程，让它跨会话也记得你的偏好和规矩
团队文档库：用 git 同步，人人本地可编辑、有完整版本历史，不依赖任何云服务

社区评价

被评为"融合了 Obsidian 和 Notion 优点的本地优先开源笔记 App":既有 Obsidian 的本地文件自由,又有 Notion 的结构化体验。来源:Medium《Tolaria: The Local-First, Open-Source Note App...》(2026-04)
评测称其为"异常精致的后来者",展示了"后发优势"——虽是同类里的迟到者,但打磨得很完整。来源:Dashen Tech《Tolaria Complete Guide 2026》
上线即有热度:登上 Hacker News(Show HN),作者也在 Refactoring 撰文称"数据涨得很猛"。来源:Hacker News、refactoring.fm
02

解决了什么痛点

为什么有人愿意花时间做这个项目？它回应了什么真实问题？

做这个项目的人，大概遇到了这些问题

用笔记软件攒知识的人,心里都有个隐忧:"我的东西到底是不是我的?" 很多笔记 App 把你的内容存成它自家的私有格式、锁在它的云上。一旦它涨价、关停、或者你想换工具,几千条笔记可能就搬不走、或者搬出来一团乱。你越用得深,越被它绑架。

第二个痛点是"AI 时代,老笔记软件不好喂给 AI"。现在大家想让 AI 帮忙整理、问答自己的知识库,但传统笔记的私有格式 AI 读起来很别扭。而纯文本文件,AI 助手能直接读写——可惜很多老牌工具不是这么设计的。

第三个痛点很个人:作者自己有 1 万多条笔记要管,现有工具没一个完全顺手。与其将就,不如自己造一个——而且每个功能都来自他真实使用中遇到的问题,不是拍脑袋加的。

所以他做的是一个"你的数据真正属于你"的知识库:笔记是纯文本、存本地、用 git 管版本、不依赖任何服务器,既给人用得顺手,又天生适合 AI 来读写。停用它你也不损失任何东西。

## Principles

- 📑 **Files-first** — Your notes are plain markdown files. They're portable, work with any editor, and require no export step. Your data belongs to you, not to any app.
- 🔌 **Git-first** — Every vault is a git repository. You get full version history, the ability to use any git remote, and zero dependency on Tolaria servers.
- 🛜 **Offline-first, zero lock-in** — No accounts, no subscriptions, no cloud dependencies. Your vault works completely offline and always will. If you stop using Tolaria, you lose nothing.
- 🔬 **Open source** — Tolaria is free and open source. I built this for myself and for sharing it with others.
- 📋 **Standards-based** — Notes are markdown files with YAML frontmatter. No proprietary formats, no locked-in data.
- 🔍 **Types as lenses, not schemas** — Types in Tolaria are navigation aids, not enforcement mechanisms. No required fields, no validation, just helpful categories for finding notes.
- 🪄 **AI-first but not AI-only** — A vault of files works very well with AI agents, but you are free to use whatever you want. We support Claude Code, Codex CLI, and Gemini CLI setup paths. We provide an AGENTS file for your agents.
- ⌨️ **Keyboard-first** — Tolaria is designed for power-users who want to use keyboard as much as possible.
- 💪 **Built from real use** — Tolaria was created to manage my personal vault of 10,000+ notes, and I use it every day. Every feature exists because it solved a real problem.

＃＃ Principles（设计原则）

📑 Files-first（文件优先）——你的笔记就是普通的 markdown 文件。它们可移植、能用任何编辑器打开、不需要「导出」这一步。你的数据属于你，而不属于任何 app。

🔌 Git-first（Git 优先）——每个 vault（知识库）都是一个 git 仓库。你能获得完整的版本历史、可以用任意 git 远程仓库、且完全不依赖 Tolaria 的服务器。

🛜 Offline-first, zero lock-in（离线优先，零锁定）——没有账号、没有订阅、不依赖云。你的 vault 完全离线可用，并且永远如此。如果你不再用 Tolaria，你不会损失任何东西。

🔬 Open source（开源）——Tolaria 免费且开源。我为自己而做，也拿来分享给别人。

📋 Standards-based（基于标准）——笔记是带 YAML frontmatter 的 markdown 文件。没有私有格式、没有被锁住的数据。如果你决定离开 Tolaria，一切都能用标准工具继续使用。

🔍 Types as lenses, not schemas（类型是「镜头」，不是「模式约束」）——Tolaria 里的「类型」是导航辅助，不是强制机制。没有必填字段、没有校验，只是帮你找笔记的有用分类。

🪄 AI-first but not AI-only（AI 优先，但不唯 AI）——一个由文件组成的 vault 和 AI agent 配合得非常好，但你也完全自由用任何你想用的东西。我们支持 Claude Code、Codex CLI、Gemini CLI 的配置路径，但你可以用任何 AI 编辑这个 vault。我们提供一个 AGENTS 文件供你的 agent 理解。

⌨️ Keyboard-first（键盘优先）——Tolaria 是为「尽可能多用键盘」的高级用户设计的。编辑器和命令面板（Command Palette）的很多设计都基于这一点。

💪 Built from real use（源于真实使用）——Tolaria 是为了管理我个人那个 1 万多条笔记的 vault 而创建的，我每天都用它。每个功能的存在，都是因为它解决了一个真实问题。

04

创新点

和同类方案比，它有什么不一样？

和常规做法不一样的地方

笔记软件不是新东西(Obsidian、Notion 都在)。Tolaria 的差异不在"又一个笔记 App",而在它把"数据彻底归你 + 为 AI 时代而生"这两件事做到了极致——笔记是纯文本、用 git 管、不依赖任何服务器,同时天生最适合让 AI 来读写。它是"后发者",但正因为后发,把这些原则想得更透。

📑
文件优先,零锁定

笔记就是纯 .md 文本,不存私有格式、不锁云端。想换工具直接拿文件走,什么都不丢。把"数据主权"还给用户,直击笔记软件最大的信任焦虑。

🔌
每个知识库都是 git 仓库

天生带完整版本历史、能用任意 git 远程备份/同步,不依赖它家服务器。把程序员的"版本管理"能力,直接变成普通笔记的底层能力。

🪄
为 AI 而生,但不唯 AI

一堆纯文本最适合 AI agent 读写——可以当"喂给 AI 的知识库"或"AI 助手的长期记忆"。支持 Claude Code/Codex/Gemini,但你用啥 AI 都行,不绑定。

🔍
"类型是镜头,不是枷锁"

很多工具的"类型/字段"是强制约束,填错就报错。它反过来:类型只是帮你找笔记的视角,没有必填、没有校验。降低记录负担,这点设计哲学很克制。

💪
"挠自己的痒"做出来的

作者自己有 1 万多条笔记、每天在用,每个功能都来自真实痛点。这种"自用打磨"出来的产品,往往比"为市场设计"的更顺手、更没废功能。

灵感卡片

看完这个项目我想到的产品点子——不一定用它的代码

灵感 1
它正好能当你的"AI 知识库底座"

你前面想做"把关注的人的内容收集成 md""给团队沉淀资料"。Tolaria 正好是放这些 md 的好地方:本地、git 备份、AI 能直接读写。可以把它当你个人/团队知识库的容器,Agent Reach 抓内容→存进 Tolaria→AI 在里面问答整理,一条线串起来。

灵感 2
"数据归你、零锁定"是最强的信任卖点

它把"你的数据属于你"当头号原则。你们做返利产品也一样:代理最怕"换平台粉丝/数据带不走"。把"数据可导出、可迁移、归你所有"明确承诺出来,是建立代理信任、降低流失的有力卖点。

灵感 3
"类型是镜头不是枷锁"的低负担设计

它不强制填字段,只把分类当辅助。你们让代理录入素材/做单信息时也可借鉴:别一上来一堆必填项吓退人,先让他随手记,分类只做"帮你找"的辅助。降低录入门槛=提高使用率。

灵感 4
"挠自己痒+真实使用"的产品方法论

给独立开发的你:作者先自用 1 万条笔记、每个功能都来自真痛点,才做成产品。最好的小产品往往先解决自己的问题。你做工具时,先问"我自己天天会用吗"。

灵感 5
"内容创作者→做工具"的个人品牌打法

作者是知名 newsletter 主理人,顺势把"自己的工作方法"做成工具,用已有读者群冷启动。这给做个人品牌的思路:你的方法论/工作流本身,可以沉淀成一个产品,用内容反哺产品。

一个不会写代码的 PM，能不能用这个项目做出东西来？

低门槛
上手难度

它是个现成桌面 App，Mac 上 brew 一行或官网下载就能装，纯用的话零代码

低门槛
依赖链

免费、本地、不用注册不用订阅；想用 git 同步/AI 读写需要一点学习

中等
Vibe Coding 适配度

作为"AI 的知识库底座"很适合；但想拿它源码改造成自己的产品，受 AGPL 协议限制

如果你要用，大概需要什么

对你来说有两种用法,门槛差很大:① 当工具直接用(强烈推荐):装上当你的个人/团队知识库,零代码,Mac 上一行命令装好。配合 AI 助手读写你的笔记,正好接上你之前"收集内容、沉淀资料"的需求。② 想抄它的代码做自己的产品:技术上可行(Tauri+React+TS),但要先过协议这关(见下方注意)。

成本:工具本身免费、本地跑不花钱。时间上当工具用 10 分钟上手;要发挥"git 同步 + AI 读写"的威力需要一点学习。

⚠️
注意（重要：协议这关）

它用的是 AGPL-3.0 协议——这是个"传染性"很强的开源协议。当工具自己用,完全没问题、免费随便用。但如果你想拿它的源码改一改、做成自己对外提供的产品(尤其是网络服务),AGPL 会要求你把你的整个产品代码也开源。所以"自用"放心用,"基于它的代码搞商业闭源产品"要非常谨慎、最好先咨询。另外它名字和 logo 受商标保护,不能直接拿来用。

07

同类项目速览

它在整个生态里处于什么位置

Obsidian 最直接对标。同样本地 markdown，但 Obsidian 闭源、强在双向链接和庞大插件生态。Tolaria 更强在原生 git 集成、AI 友好、开源。你甚至能拿 Tolaria 直接打开已有的 Obsidian 笔记库并行用。
Notion 强在结构化数据库和团队协作，但数据存在它云上、是私有格式。Tolaria 主打"本地、纯文本、归你"，被评测说成"融合了 Obsidian 与 Notion 优点"。
PM Brain（你之前看过） 思路同源:都是"本地 markdown 文件夹 + AI 读写"的第二大脑。区别是 PM Brain 是给 AI 用的轻量约定，Tolaria 是个有完整界面的桌面 App。可理解为同一理念的不同形态。
Logseq / 各类本地笔记 同属"本地优先、开源"阵营。Tolaria 的差异化卖点是 git-first + AI-first + 后发把体验打磨得更精致。
📌
这个项目适合：

在意"数据归自己"的重度笔记用户;想给 AI 配一个本地知识库/长期记忆的人;喜欢键盘操作的效率党;以及做个人知识管理的独立开发者。


不适合：

需要多人实时协作、网页端随处可用的团队(那是 Notion 的强项);完全不想碰 git/本地文件、只要开箱即用云笔记的轻度用户。
