这是什么？

一句话:给你的 AI 装上一双"能看整个互联网"的眼睛。装上它,你的 AI 助手就能去读推特、搜 Reddit、看 YouTube 字幕、刷小红书、扒 B站视频、读任意网页和 RSS——而且全程免费、不用付那些贵得离谱的平台 API 费。

它最聪明的地方是:不自己重写一堆抓取代码,而是当一个"总管"——帮你从一堆开源小工具里选出当下最稳的那个、自动装好、还定期体检。平台一改反爬规则、某个工具失效了,它替你换成能用的下一个,你完全无感。对你来说,这正好补上了你那个"定时收集关注的人内容"项目里最难啃的"怎么合规拿到数据"那一环。作者是国内开发者,README 全中文。

给 AI 装眼睛：让 AI 免费读取 X/YouTube/小红书/B站/Reddit 等多平台内容

解决了什么痛点

为什么有人愿意花时间做这个项目？它回应了什么真实问题？

做这个项目的人，大概遇到了这些问题

这个痛点你最近刚亲身体会过:AI 能帮你干很多活,但你让它"上网帮我看看",它就抓瞎。让它看 YouTube 教程→拿不到字幕;搜推特→API 要付费;上 Reddit→IP 被封 403;看小红书→必须登录打不开;扒 B站→被风控拦死;读网页→抓回一堆乱码标签。每个平台都有自己的门槛。

第二个痛点是"不是做不到,是每个都要自己折腾"。要付费的 API、要绕过的封锁、要登录的账号、要清洗的数据——你得一个平台一个平台去踩坑、装工具、调配置,光让 AI 能读个推特就得折腾半天。这正是你那个"四平台收集"项目卡住的地方。

第三个痛点更隐蔽:"平台规则一直在变,今天能用明天就挂"。某个抓取小工具停更了、某个平台升级了反爬,你辛苦配好的链路说断就断,又得重新找替代。普通人根本盯不过来。

所以作者做的不是"又一个抓取工具",而是一个"能力总管":把"当下最稳的接入方式"替你选好、装好、体检好,平台变了他来修、有新渠道他来加,你只管用。一句话给 AI 就装好,这就是它的全部价值。

为什么需要 Agent Reach？

AI Agent 已经能帮你写代码、改文档、管项目——但你让它去网上找点东西，它就抓瞎了：

📺「帮我看看这个 YouTube 教程讲了什么」→ 看不了，拿不到字幕
🐦「帮我搜一下推特上大家怎么评价这个产品」→ 搜不了，Twitter API 要付费
📖「去 Reddit 上看看有没有人遇到过同样的 bug」→ 403 被封，服务器 IP 被拒
📕「帮我看看小红书上这个品的口碑」→ 打不开，必须登录才能看
📺「B站上有个技术视频，帮我总结一下」→ 拿不到，通用下载工具被 B站风控全面拦截
🔍「帮我在网上搜一下最新的 LLM 框架对比」→ 没有好用的搜索，要么付费要么质量差
🌐「帮我看看这个网页写了啥」→ 抓回来一堆 HTML 标签，根本没法读
📦「这个 GitHub 仓库是干嘛的？Issue 里说了什么？」→ 能用，但认证配置很麻烦
📡「帮我订阅这几个 RSS 源，有更新告诉我」→ 要自己装库写代码

这些不难实现，但是需要自己折腾配置。每个平台都有自己的门槛——要付费的 API、要绕过的封锁、要登录的账号、要清洗的数据。

Agent Reach 把这件事变成一句话：把安装文档链接复制给你的 Agent，几分钟后它就能读推特、搜 Reddit、看 YouTube、刷小红书了。已经装过了？更新也是一句话。

作者注：⭐ Star 这个项目，会持续追踪各平台的变化、接入新的渠道。平台封了我们修，有新渠道我们加。

✅ 在你用之前，你可能想知道

💰 完全免费：所有工具开源、所有 API 免费。唯一可能花钱的是服务器代理（$1/月），本地电脑不需要。
🔒 隐私安全：Cookie 只存在你本地，不上传不外传。代码完全开源，随时可审查。
🔄 持续换代：每个平台都是「首选 + 备选」多后端路由。某个接入方式失效了就换下一个，你无感（2026-06 实例：yt-dlp 被 B站风控封死 → 已切换 bili-cli，用户零操作）。
🤖 兼容所有 Agent：Claude Code、OpenClaw、Cursor、Windsurf……任何能跑命令行的 Agent 都能用。
🩺 自带诊断：agent-reach doctor 一条命令告诉你哪个通、哪个不通、怎么修。

支持的平台

（表格：平台 ｜ 装好即用 ｜ 配置后解锁 ｜ 怎么配）

🌐 网页：阅读任意网页 ｜ 无需配置
📺 YouTube：字幕提取 + 视频搜索 ｜ 无需配置
📡 RSS：阅读任意 RSS/Atom 源 ｜ 无需配置
🔍 全网搜索：全网语义搜索 ｜ 自动配置（MCP 接入，免费无需 Key）
📦 GitHub：读公开仓库+搜索；配置后解锁私有仓库、提 Issue/PR、Fork ｜ 告诉 Agent「帮我登录 GitHub」
🐦 Twitter/X：读单条推文；解锁搜索推文、浏览时间线、读长文 ｜「帮我配 Twitter」
📺 B站：搜索+视频详情（bili-cli，无需登录）；解锁字幕（OpenCLI）｜「帮我配 B站」
📖 Reddit：无零配置路径（匿名接口已被封）；解锁搜索+读帖和评论 ｜ 桌面装 OpenCLI 用浏览器登录态，或 rdt-cli + Cookie
📕 小红书：解锁搜索、阅读、评论 ｜ 桌面装 OpenCLI（刷过小红书即可用）；服务器用 xiaohongshu-mcp 扫码
💼 LinkedIn：Jina Reader 读公开页；解锁 Profile/公司页/职位搜索 ｜「帮我配 LinkedIn」
💻 V2EX：热门帖、节点帖、帖子详情+回复、用户信息 ｜ 无需配置
📈 雪球：股票行情、搜索股票、热门帖、热门股票排行 ｜「帮我配雪球」
🎙️ 小宇宙播客：播客音频转文字（Whisper 转录，免费 Key）｜「帮我配小宇宙播客」

作者注：不知道怎么配？不用查文档，直接告诉 Agent「帮我配 XXX」，它会一步步引导。需要 Cookie 的平台（Twitter、小红书等）优先用 Chrome 插件 Cookie-Editor 导出 Cookie 发给 Agent。Cookie 只存本地。本地电脑不需要代理，代理只在部署到服务器时才需要（~$1/月）。

快速上手

⚠️ OpenClaw 用户请先确认 exec 权限已开启（Agent Reach 依赖 Agent 执行 shell 命令）：openclaw config set tools.profile "coding"，重启 Gateway 后开新对话。其他平台（Claude Code、Cursor、Windsurf 等）不受此限制。

复制这句话给你的 AI Agent：帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md。就这一步，Agent 会自己完成剩下所有事情。已安装过？更新也是一句话。担心安全？可用安全模式（加 --safe 参数，不自动装系统包，只告诉你需要什么）。

它会做什么（原文折叠区）：① pip install 装好 agent-reach 命令行（自带 yt-dlp、feedparser）；② 自动检测并安装 Node.js、gh CLI、mcporter；③ 通过 MCP 接入 Exa 搜索（免费无需 Key）；④ 判断本地还是服务器并给配置建议；⑤ 在 Agent 的 skills 目录注册 SKILL.md，以后遇到「全网调研/搜推特/看视频」会自动知道调哪个工具；⑥ 默认只激活 6 个零配置渠道，小红书/Twitter/Reddit 这些需登录的会列菜单问你、点名才装。装完 agent-reach doctor 报告每个渠道状态。

装好就能用

不需要任何配置，告诉 Agent 就行：「帮我看看这个链接」→ curl https://r.jina.ai/URL；「这个 GitHub 仓库做什么」→ gh repo view；「这个 YouTube 视频讲了什么」→ yt-dlp 提取字幕；「B站搜一下 AI 教程」→ bili search（无需登录）；「全网搜 LLM 框架对比」→ Exa 语义搜索；「订阅这个 RSS」→ feedparser 解析。

不需要记命令。Agent 读了 SKILL.md 之后自己知道该调什么。需要登录的平台（小红书、Twitter、Reddit），告诉 Agent「帮我配 XXX」即可解锁。

能力边界：读内容 vs 操作网页

有些任务超出「读」的范围：登录后的网页操作、表单提交、多账号隔离、并行浏览器会话、自动化流程中的登录/验证/风控提示等高摩擦环节的人工接手。这类「动手」场景，可以配合 BrowserAct 这类浏览器自动化工具（30+ 预制平台技能，支持 Claude Code / OpenClaw / Cursor 等）。

设计理念

Agent Reach 是一个能力层（capability layer），不是又一个工具。它比任何具体实现高一层——负责选型、安装、体检、路由，不负责底层读取本身。读取由 Agent 直接调用上游工具完成，没有包装层。

你给新 Agent 装环境时，总要花时间找工具、装依赖、调配置——Twitter 用什么读？Reddit 怎么登录？小红书的 CLI 停更了换什么？Agent Reach 做的事很简单：当下最稳的接入方式，替你选好、装好、体检好。接入方式会换代（2026 年 3 月一批单平台 CLI 集体停更，作者换了路由），你不用操心。

🔌 每个平台 = 首选 + 备选的有序后端列表。换接入方式 = 调整列表顺序，不是重写代码。agent-reach doctor 会告诉你每个平台当前在用哪个后端。（原文给了 channels/ 目录结构，如 twitter.py → twitter-cli ▸ OpenCLI ▸ bird；bilibili.py → bili-cli ▸ OpenCLI ▸ 搜索 API 等。）每个渠道文件按序真实探测各候选后端，第一个完整可用的当选，坏掉的会给修复处方。

当前选型（部分）：读网页→Jina Reader；读推特→twitter-cli（备选 OpenCLI）；Reddit→OpenCLI（备选 rdt-cli）；YouTube 字幕+搜索→yt-dlp（已不再用于 B站）；B站→bili-cli（yt-dlp 被 B站风控 412 封死）；搜全网→Exa via mcporter；GitHub→gh CLI；RSS→feedparser；小红书→OpenCLI（备选 xiaohongshu-mcp ▸ xhs-cli）；LinkedIn→linkedin-scraper-mcp（备选 Jina Reader）。作者注：这些都是「当前选型」，基于真机实测定期复核。

安全性

🔒 凭据本地存储：Cookie、Token 只存本机 ~/.agent-reach/config.yaml，文件权限 600，不上传不外传。
🛡️ 安全模式：install --safe 不自动改系统，只列需要什么。
👀 完全开源：代码透明可审查，依赖工具也都开源。
🔍 Dry Run：install --dry-run 预览所有操作不做改动。
🧩 可插拔架构：不信任某组件？换掉对应 channel 文件即可。

🍪 封号风险提醒：使用 Cookie 登录的平台（Twitter、小红书等），通过脚本/API 调用存在被平台检测并封号的风险。请务必使用专用小号，不要用主账号——一是封号风险，二是 Cookie 等同完整登录权限、小号能在泄露时限制影响范围。

安装方式：一键全自动 install --env=auto（个人电脑）；安全模式加 --safe（生产/共用机）；仅预览加 --dry-run。卸载：agent-reach uninstall（清除配置、skill 文件、MCP 配置；可 --keep-config 保留 token）。

常见问题 / FAQ（节选）

AI Agent 怎么免费搜索 Twitter/X？用 twitter-cli 通过 Cookie 认证访问，完全免费：pipx install twitter-cli，确保浏览器已登录 x.com，即可 twitter search "关键词"。

Reddit 返回 403 怎么办？Reddit 所有访问都需登录态。桌面首选 OpenCLI（登录过 reddit.com 即可 opencli reddit search）；备选 rdt-cli。中国大陆访问 Reddit 需代理。

怎么让 AI 读小红书？桌面首选 OpenCLI——复用浏览器登录态，平时刷过就能用、零配置；之后 opencli xiaohongshu search/note。服务器用 xiaohongshu-mcp（无头浏览器扫码）。

免费吗？有 API 成本吗？100% 免费，所有后端都是开源工具、不需付费 Key。唯一可选成本是住宅代理（~$1/月），仅当你的网络封锁 Reddit/Twitter（如中国大陆）时才需要。

兼容 Claude Code/Cursor/OpenClaw/Windsurf 吗？是的，任何能跑 shell 命令的 AI 编程 agent 都能用。

创新点

和同类方案比，它有什么不一样？

和常规做法不一样的地方

别人做抓取,是自己写一套去抠每个平台的数据;它的思路完全不同——"我不自己抓,我帮你管"。它当一个"能力总管/中间层",从一堆现成开源小工具里选当下最稳的、装好、体检、出问题自动换路。这个"高一层"的定位,是它能扛住"平台天天变反爬"的关键。

🔀
"首选+备选"多后端路由

每个平台都配一串"能用的工具列表",当前这个被封了就自动切下一个(B站 yt-dlp 被封→秒切 bili-cli,用户零操作)。换路=调顺序,不用重写。这是它最硬的创新。

💸
零 API 费用

全靠开源工具 + Cookie 登录态,绕开了各平台动辄上千的官方 API 费。对要长期、批量抓内容的人,这是真金白银的省。

🗣️
"一句话"安装与配置

不用你看文档:把一个链接发给 AI 就自动装好;要配某平台就说"帮我配 XXX",AI 一步步引导。把技术门槛压到了"会聊天就会用"。

🩺
自带"体检"诊断

一条 doctor 命令告诉你每个平台通不通、现在走哪条路、坏了怎么修。把"黑箱抓取"变成"看得见状态",这点对非技术人特别友好。

🌏
国产平台覆盖齐

小红书、B站、雪球、V2EX、小宇宙播客都覆盖——这是国外同类工具几乎不碰的。对你做中文内容收集,这点非常稀缺、非常对路。

灵感卡片

灵感 1 ⭐
它就是你"四平台收集"项目缺的那块拼图

你之前卡在"怎么合规拿到 X/YouTube/小红书/B站的内容"——这个项目正好把这一层做了,还免费、四个平台全覆盖。建议直接拿它当你那个项目的"抓取层",你只要在它之上做"定时+整理成给代理看的日报"就行。

灵感 2
"首选+备选自动切换"的容错思路

它每个平台配一串备选、坏了自动切。你们做转链也可以借鉴:某个平台转链接口抽风时,自动切到备用通道,别让运营卡在"转不出来"。把"单点会挂"变成"多路兜底"。

灵感 3
"一句话给AI就装好"的极简交付

它把复杂安装压缩成"发个链接给 AI"。你们给代理的工具/教程也能学:别让代理读长文档,做成"把这句话发给助手,它带你一步步配好",门槛越低留存越高。

灵感 4
"体检诊断"做成代理工具的标配

它用 doctor 一键告诉你哪通哪不通。你们 App 也可以给代理一个"一键自检":微信授权有没有过期、转链通不通、素材同步没——主动暴露问题,比等代理来报障好。

灵感 5
"能力总管"的产品定位思路

它不抢着做底层,而是做"帮你选好、管好"的中间层。这给独立开发的你一个定位启发:在一个变化快、工具多的领域,"帮人选型+兜底维护"本身就是高价值产品,不一定要自己造轮子。
