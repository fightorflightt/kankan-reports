#用AI做幻灯片 #看图选风格 #零依赖单文件 #Claude Skill #反AI塑料感

这是什么？

一句话：一个让"不会写代码、也讲不清自己想要什么风格"的人，靠"看图选一选"就能做出一套高级网页幻灯片的 AI 技能。

它不是又一个 PPT 模板库。你打开 AI 编程助手（比如 Claude Code），跟它说"我要做一份 AI 创业的融资路演 PPT"，它会先问你内容，然后当场生成 3 个不同风格的封面让你挑——你不用懂设计、也不用形容"我想要简约高级风"，看哪张顺眼点哪张，它就照着那个风格把整套幻灯片做出来，最后直接在浏览器里打开。

做出来的东西是一个单独的网页文件（HTML），不依赖任何软件、不怕版本过期、手机电脑都能开，还能一键发成网址分享、或导成 PDF。作者是 Zara Zhang（@zarazhangrui），项目本身这份介绍页，就是用它自己做出来的。

应用场景

创业者赶融资路演：手里只有一堆零散想法和数据，没时间也没设计师，让 AI 当晚就出一版不像"模板货"的 pitch deck。
要上台做技术/产品分享：写惯了文档但不会排版，把要讲的要点丢给它，挑个风格就拿到一套能投屏、带动画的演示。
救一份丑 PPT：手上有个老旧的 PowerPoint（.pptx），让它把里面的文字和图片全抽出来，换成一套好看的网页版重做。
讲不清自己要什么风格的人：脑子里没有"想要什么样"，但一看就知道"喜不喜欢"——靠它生成的预览图来回挑，挑出自己的审美。

解决了什么痛点

为什么一个人愿意花时间反复迭代几十次、把它做成一个技能？

做幻灯片的人，其实卡在这三件事上

第一，"我知道丑，但我说不出想要什么样的好。" 大多数人不是设计师，让你形容"我想要的风格"，你只能憋出"高级一点、简约一点、别太花"这种没法落地的词。市面上的 AI 做 PPT 工具偏偏都要你先用文字描述需求——这等于把最难的一步甩回给了不会描述的人。这个项目反过来：它不问你想要什么，直接做几个样子给你看，你只需要回答"喜欢哪个"。看图选择，比凭空描述容易一百倍。

第二，"AI 做出来的东西，一眼就是 AI 做的。" 现在用 AI 生成的页面有种通病——紫色渐变、白底、圆角卡片，千篇一律的"科技塑料感"（作者原话叫 AI slop，AI 流水货）。投出去大家一看就知道你是随手生成的，不走心。这个项目专门跟这种"通用感"对着干，内置了一批精心调过的、各有强烈个性的设计风格，就是为了让你的东西看起来像"专门做的"，而不是"批量生成的"。

第三，"今天能打开的文件，过两年就打不开了。" 用各种花哨框架做的东西，往往过一阵子环境一变就跑不起来，沦为技术债。作者的信念很朴素：一个单独的 HTML 网页文件，十年后照样能打开——不挑软件、不挑设备、不用安装任何东西，手机点开就能放映。这让"做出来的成果"真正属于你，而不是绑死在某个工具里。

### Key Features

- **Zero Dependencies** — Single HTML files with inline CSS/JS. No npm, no build tools, no frameworks.
- **Visual Style Discovery** — Can't articulate design preferences? No problem. Pick from generated visual previews.
- **PPT Conversion** — Convert existing PowerPoint files to web, preserving all images and content.
- **Anti-AI-Slop** — Curated distinctive styles that avoid generic AI aesthetics (bye-bye, purple gradients on white).
- **Bold Template Pack** — Optional design-forward templates from `beautiful-html-templates`, loaded progressively so safe presets still work as the default fallback.
- **Production Quality** — Accessible, fixed 16:9, well-commented code you can customize.

Install directly from this public GitHub repo. Run these as two separate Claude Code messages; do not paste both lines into the prompt at once.

/plugin marketplace add https://github.com/zarazhangrui/frontend-slides

### Bold Template Pack

The skill also includes 34 optional bold design systems from `beautiful-html-templates`, such as **Neo-Grid Bold**, **Editorial Tri-Tone**, **Creative Mode**, **Broadside**, **Signal**, and **Vellum**.

During style discovery, the preview set is:

- 1 safe preset from `STYLE_PRESETS.md`
- at least 1 bold template option from `bold-template-pack/selection-index.json`
- 1 wildcard option, either another bold template or a self-generated custom design

The agent reads the compact bold template index first, then loads only the shortlisted candidates' small `preview.md` cards for title-slide previews. It loads the full `design.md` for exactly one bold template only after the user picks that template for the final deck. If the user picks a custom wildcard, the agent expands that preview's own CSS and layout system into the full deck.

## Sharing Your Presentations

After creating a presentation, the skill offers two ways to share it:

### Deploy to a Live URL

One command deploys your slides to a permanent, shareable URL that works on any device — phones, tablets, laptops:

```bash
bash scripts/deploy.sh ./my-deck/
# or
bash scripts/deploy.sh ./presentation.html
```

Uses [Vercel](https://vercel.com) (free tier). The skill walks you through signup and login if it's your first time.

### Export to PDF

Convert your slides to a PDF for email, Slack, Notion, or printing:

```bash
bash scripts/export-pdf.sh ./my-deck/index.html
bash scripts/export-pdf.sh ./presentation.html ./output.pdf
```

Uses [Playwright](https://playwright.dev) to screenshot each slide at 1920×1080 and combine into a PDF. Installs automatically if needed. Animations are not preserved (it's a static snapshot).

创新点

和别的"AI 做 PPT"工具比，它真正不一样的地方在哪

和常规做法不一样的地方

市面上"AI 生成 PPT"的工具一大把，但它们大多是"你描述需求 → AI 套模板填内容"。Frontend Slides 的不同，不在于"AI 会做 PPT"这件事，而在于它重新设计了"人和 AI 一起定风格"的交互方式，以及它对"成果应该是什么形态"的坚持。下面几点是它真正的创新。

👁️
把"描述风格"换成"看图选择"

这是它最核心的产品洞察。它不再逼你说清"我想要什么风格"（这恰恰是普通人做不到的），而是当场做 3 张不同风格的封面让你挑。对应到你熟悉的语言：这是把一个"开放式问答"改成了"选择题"——决策门槛瞬间降到地板。做单时给运营选商品图风格、给用户选样式，这套"少说、多给样子挑"的思路完全可以借鉴。

🧩
"渐进式披露"：先给地图，再按需翻页

它没有把所有模板、规则、动画代码一股脑塞给 AI，而是让 AI 先看一张"流程地图"，只有当你选了某个风格，才去加载那一个风格的完整细节。好处是 AI 不会被海量信息淹没、出错率更低、反应更快。这是"少即是多"在 AI 工具设计上的体现，也是为什么它能稳定产出而不跑偏。

🚫
把"反 AI 塑料感"做成产品卖点

大多数 AI 工具追求"能用就行"，它却专门跟"一眼假的 AI 通用感"较劲——内置 12 个精选预设 + 34 个个性强烈的模板，每个都有独立审美。它把"高级感/不像批量货"本身当成了核心价值。对靠内容和视觉抓人的私域运营来说，这种"反流水货"的取向特别有共鸣。

📦
成果是"一个能传十年的单文件"

它坚持产出单个 HTML 网页文件，不绑定任何软件、不依赖任何在线服务。这意味着你的成果是真正"拿在自己手里"的资产：发微信、发网址、导 PDF、十年后再打开都行。这跟那些"做完只能锁在某个 App 里"的工具是根本性的区别。

🛠️
它本身就是"把工作流变成技能"的范例

作者公开讲过做法：先让 AI 反复打磨一个像幻灯片的网页、迭代几十次到满意，再把这套"什么算好幻灯片"的经验和流程，固化成一个可复用的 skill。这其实给了所有人一个模板——你自己反复在做的某件事，也可以这样"沉淀成一个 AI 技能"，下次一键复用。

灵感卡片

看完这个项目，我想到的几个能用在咱们产品/运营上的点子——不一定要用它的代码

灵感 1
做单"选风格"也改成看图挑，别让运营描述

运营做商品图时，最难的就是"这次想要什么调性"说不清。可以借它的思路：在做单工具里预置几套商品图模板风格（喜庆促销、简约高级、母婴温馨…），运营只需点封面挑一个，系统就把这套风格套到当前商品上。把"开放式审美决策"变成"三选一"，新手运营也能出稳定好看的图。

灵感 2
用它给代理/运营做"招商培训"演示稿

咱们经常要给新代理讲"怎么做单、怎么赚佣金"。与其发一堆文字，不如用这个 skill 快速生成一份好看的网页版讲解稿，发个网址链接到微信群，手机点开就能划。比 PPT 更轻、比文档更好看，还能随时改完重发。低成本、立刻能试。

灵感 3
"把反复做的事沉淀成一个技能"——值得我们自己干

这个项目最大的启发其实是方法论：作者把"我反复在做的一件事"固化成了一键可复用的 AI 技能。咱们运营里有很多重复动作（写群文案、做下单步骤图）。可以挑一个最高频的，照这个套路沉淀成内部小工具，让全团队一键复用、质量统一。这跟你已经在做的 /kanxia、凑价引导这类"自己造工具"的路子是一脉相承的。

灵感 4
"成果是个能传的单文件网址"这件事本身很香

它产出的不是锁在 App 里的东西，而是一个能直接发微信、手机秒开的网址。咱们很多要发给用户/代理的东西（活动说明、爆品榜单、玩法介绍）都可以走这条路：做成轻量网页，一个链接甩群里，比图片清晰、比长文好看、还能随时更新内容而链接不变。

灵感 5
"反流水货"是私域内容的隐形竞争力

它把"别让人一眼看出是 AI 批量生成的"当成核心价值。私域运营同理：群里千篇一律的模板图会让用户疲劳、信任感下降。值得我们在做单素材上刻意追求一点"专属感、用心感"——哪怕只是换几套有辨识度的视觉模板轮着用，也比天天同一个套路更抓人。

Vibe Coding 适配度

极高。它天生就是为"跟 AI 对话把事办了"设计的——你只管说内容、挑风格，剩下全交给 AI，正是你最舒服的协作方式。

如果你要用，大概需要什么

时间：第一次把环境装好（Claude Code + 这个插件）大概半小时到一小时；之后做一份幻灯片，从开口到出成品，十几分钟级别。

钱：工具本身免费（MIT 协议，随便用）。真正的成本是你用的那个 AI 助手的订阅费/调用费。部署网址用的 Vercel 有免费档，导 PDF 也免费。

前置知识：核心使用不需要任何代码知识。唯一会卡住非技术用户的，是"安装"那一关——要在电脑上装 Claude Code、跑两条命令。这一步建议拉一个懂技术的人帮你配一次，配好之后你自己用完全没问题。

人：你一个人就能用起来，不需要开发团队。

⚠️
注意几个隐性成本/坑

① 它不是一个 App、也不是网站，没有图形界面——你得通过命令行工具（Claude Code 这类）来用，对纯产品/运营同学，第一次接触会有点劝退，建议找人带一次。

② 它做的是"网页式幻灯片"，不是真正的 .pptx 文件。如果对方非要可编辑的 PowerPoint 源文件，它给不了（只能导出静态 PDF，且导出后动画会丢失）。

③ "转 PPT"功能对环境要求更高（要 Python 环境），比"从零做"更容易卡住。

④ 装的是别人写的 AI 技能，本质是让 AI 按一份脚本在你电脑上跑命令。虽然它通过了第三方安全扫描，但养成习惯：来路不明的 skill 别乱装。

"用代码/AI 做演示"这条赛道上，它处在什么位置

Gamma / 闪击 PPT 等 AI PPT 工具 最直接的竞品方向：在线网站，输入主题就生成 PPT，有图形界面、上手最简单。但本质是"套模板"，做出来的东西更容易"撞脸"，且成果绑在它们平台里。Frontend Slides 反过来——更个性化、成果是你自己的单文件，但门槛高（要会用命令行 AI 助手）。
Slidev · 30k+ stars 面向程序员的"用 Markdown 写 PPT"框架。功能强、生态成熟，但要自己写代码和配置，纯小白用不了。Frontend Slides 把"写"换成了"跟 AI 说 + 看图挑"，对非技术用户友好得多。
reveal.js · 67k+ stars 元老级的"纯网页做幻灯片"框架，最久最成熟。同样需要手写 HTML，是给开发者用的底层工具。Frontend Slides 某种意义上是"给这类底层能力套了一层 AI 交互"，让不写代码的人也能享受到网页幻灯片的好处。
PowerPoint / Keynote / 各类在线模板站 传统主力。胜在所见即所得、可精细手调、生态无敌。但要好看就得有设计功底或花钱买模板，且做出来还是"模板感"。Frontend Slides 主打的正是"不用设计功底也能不像模板货"。
📌
这个项目适合：

已经在用（或愿意上手）Claude Code 这类 AI 编程助手的人；看重"成果好看、不撞脸、能长期保存和自由分享"的个人或小团队；讲不清自己审美、但"一看就知道喜不喜欢"的非设计师。


不适合：

完全不想碰命令行、只想打开一个网站点几下的纯小白（除非有人帮你配好环境）；必须交付可编辑 .pptx 源文件的场景；需要在演示里做超复杂图表/精细动效逐帧打磨的专业设计需求。
