这是什么？

一份「AI 时代普通人怎么搞副业赚钱」的信息黄页。作者自己就是靠 AI 内容起家的实践者，把全网验证过的变现路径——从 ChatGPT 套壳站到 AI 表情包、从声音克隆到无人直播——按方向分门别类整理成一张大地图，每条路径都附带教程链接。

它不是一个「产品」，而是一本已经帮你筛选过的灵感手册。17.5k Star 说明了大批人对「用 AI 赚钱」这件事的信息饥渴。

02

解决了什么痛点

为什么有人愿意花时间做这个项目？它回应了什么真实问题？

做这个项目的人，大概遇到了这些问题

痛点一：信息差就是钱，但信息太散了。2023年 ChatGPT 爆火后，中文互联网上关于「怎么用 AI 赚钱」的内容井喷——抖音博主在讲、即刻网友在晒、推特 KOL 在吹，但都是碎片化的。一个普通人想知道「我能用 AI 干什么副业」，需要刷几十个平台，还分不清哪些是割韭菜哪些是真能搞。作者做的就是把散落全网的信息收纳进一个地方，按变现路径归类，省去普通人搜集筛选的时间。

痛点二：知道 AI 很强，但不知道怎么变现。很多人会用 ChatGPT 聊天、用 Midjourney 画图，但这两件事怎么变成银行卡里的数字？作者用自己做网站从 0 到 2 万 DAU 的真实经历告诉你：做内容搞流量 + 卖铲子（贴牌/分销）。不是空谈理论，而是给了具体到「小红书 AI 头像壁纸一个月日入 1000」「用 GPT 批量写小红书文案篇篇爆款」这种颗粒度的案例。

痛点三：AI 信息圈鱼龙混杂，韭菜太多。作者自己也承认，他拒绝做社群因为「觉得很割韭菜」，后来大量仿站蜂拥而上，韭菜社群遍地。他的解法是把信息开源——17.5k Star 的仓库，所有人免费看，内容透明。与其被收智商税，不如直接看一个已经验证过可行性的路线图。

README 原文精读

本项目 README 为中文撰写，以下为核心章节摘录与产品视角解读

关于这个仓库的故事

22年末ChatGPT到来，我开始使用AI，23年初正好为了实践一个SEO工具+WP导航站模板，正愁不知道做什么内容，想了想反正自己也在学习AI，或许有机会，就用费曼学习法的方式把一些基本资料做到了网站里。策略也比较简单，因为学习是一个漏斗，所以做大众最需要的就可以。

这个网站最早是发在即刻了，后来不知道怎么推特的A姐也转发了，然后就在抖音上开始传播。传播机制也很简单，因为内容对用户有价值，抖音上的博主就自发去发视频晒这个网站……两周DAU到了2万。

那时候一个礼拜就能拉一万的AI私域社群，变现也比较简单，卖账号，挂ChatGPT套壳分销。

产品解读

作者的起盘路径非常清晰：费曼学习法做内容 → 即刻冷启动 → KOL 转发 → 抖音博主自发传播 → 三方共赢（博主得流量、用户得工具、作者得流量+私域）→ DAU 2万。这不是技术驱动，是纯内容运营驱动。他的策略「做大众最需要的」抓住了漏斗最宽的那一层：怎么注册、怎么用提示词。到 2023 年底 AI 内容站泛滥后，他转而把垂直副业内容打包成 GitHub 仓库，被知名开发者宝玉转发，DAU 又回到 1 万。

四条建议
  • 做内容搞流量，而不是做一个产品去卖 — 搞来流量，流量自然需要产品，找到别人的产品，合作促成交，就可以赚到钱。
  • 做铲子，而不是挖金子 — 挖金子对项目的全面要求很高，而且风险大，卖铲子往往会更直接。贴牌、分销。
  • 不想花钱，就要花时间 — 花钱跟人买认知，然后去执行，风险小但需要付出成本。
  • 做资产，而不是卖时间 — 如果你的时间不能同时卖出多份，那和打工没差别。
产品解读

这四条建议是整套内容的方法论底座，每一条都值得展开：「做铲子不挖金子」= 做基础设施/工具比做终端产品风险小；「做资产不卖时间」= 追求可复利、可复用的产出（内容、代码、品牌），不要陷入定制化服务。这套逻辑不仅适用于 AI 副业，放在任何互联网变现场景都成立。

技术赚钱的一些认知
  • 做垂类，而不是做平台 — 增加一个垂类，你就能被人所知道。
  • 做顺人性的，而不是逆人性的 — 做小说就顺人性，搞教育就逆人性。
  • 从小切入 — 别一上来想搞太大。
  • 开源赚流量，比闭源挖金子强
  • Build in Public — 这会给你带来极大的公共认同。
  • Copycat — 欧美到国内，国内到非洲，有信息茧房就有钱赚。
  • 做国外而非国内 — 赚美金好一些，国内有各种LLM审核问题。
  • 赚有钱人的钱，赚想赚钱人的钱
产品解读

这是面向程序员的产品经理课浓缩版。「Copycat」策略本质上就是跨市场套利——把一个市场验证过的模式搬运到另一个信息不对称的市场；「开源赚流量」是典型的 PLG（产品驱动增长）策略，用开源内容建立信任和流量池，再靠闭源产品或服务变现。

已验证的技术赚钱方案
  • 套壳站：GPT/MJ 套壳站，做成 SaaS 或支持私有化部署贴牌，几千块一套卖给小老板。
  • API 聚合：把各类 LLM 缝合成 API，中间商赚差价。
  • 发卡站：开源发卡站自动卖 GPT 账号。
  • 聚焦的 AI 工具：AI 写论文、AI 证件照，把 AI 聚焦到某个领域的刚需。
  • 模型训练：帮甲方做模型训练，但不好交付。
产品解读

这五个方向在 2023 年都真实有人赚到钱了。套壳站和 API 聚合本质是「搬运工」生意——门槛低、竞争激烈，先发优势很重要。聚焦 AI 工具才是长期方向：刚需 + 垂直 + 低替代性。作者也指出模型训练「不好交付」——这个是真实踩坑经验，国内模型效果可能达不到甲方预期。

AI 自媒体赚钱方向（目录摘录）
  • AI 脚本视频：图片绘本故事、虚拟人口播、小说漫画推文、电影剧情解说、热点选题
  • 视频变幻：瞬息全宇宙转场、热舞小姐姐动画、无限穿越放大缩小
  • 视频翻译搬运：跨语言搬运、自动翻译配音
  • AI 图片：个人头像、桌面壁纸、模特换装、商品广告、儿童绘本、表情包、家具装修、LOGO、照片修复
  • AI 文案：新媒体推文、论文代写、小说编剧、简历改写
  • AI 音频：声音克隆、AI 音乐（Suno / Stable Audio）
  • AI 直播：无人货架直播、虚拟人直播
产品解读

这是 README 篇幅最大的部分——每个子方向下列出 4 条具体教程链接。覆盖面极其广，但深度不均：热门方向（头像、壁纸、绘本）教程多且详细，冷门方向（家具装修、简历改写）相对薄弱。整体看下来，AI 图片方向是案例最多的——因为 Midjourney/Stable Diffusion 的「出图效果」在短视频平台有天然传播优势。

项目现状（作者自述）

这个仓库目前已经作废……后来网站就没继续运营了，结果今天在推上一个用户分享了这个仓库，我看评论区很多人说跳转黄色网站了，我就来把这个仓库处理一下吧。

很多时候我很佩服那些做黑产的，他们不论是在做自动化思路上，还是在做流量上，都有很多创新。比如这个仓库的域名被他们收走了，然后去做截流。我估计他们应该有脚本监听各类Repo去买无法解析的域名做这个事情的。

产品解读

作者用亲身经历展示了一个残酷现实：AI 副业圈不仅卷，还有黑产。仓库域名被灰产抢注做截流，说明这个赛道的信息差本身就是一种可以被劫持的资产。作者对黑产的评价很有意思——「佩服他们在自动化和流量上的创新」——这是一种产品人的冷静观察：坏人往往最懂增长。

作者近况

产品解读

作者转型方向很有参考价值：从「AI 副业信息资源」转向「AI 营销工具」——从卖地图到卖铲子的进化。xaicreator 是 typefully（一个海外 Twitter 写作工具）的 copycat，再次印证了他自己说的「Copycat 策略」。

04

创新点

和同类方案比，它有什么不一样？

和常规 AI 教程不一样的地方

市面上的 AI 副业教程大多分两种：一种是「月入十万」的标题党付费课，只给概念不给路径；一种是零散的 YouTube/B站教程，看完不知道下一步。这个仓库的差异化在于：作者自己就是跑通这条路的玩家，他不是在教你，而是在记录自己验证过的东西。

💰
从信息喂饭到方法论输出

大多数「AI 工具导航站」只是罗列链接。这个仓库在链接之上，给出了作者自己总结的方法论——「做铲子不挖金子」「Copycat」「开源赚流量」——这些都是他从自己网站 DAU 从 0 到 2 万再到衰落的全过程中提炼的。信息会过时，方法论不会。

🔄
GitHub 作为内容分发渠道

用 GitHub 仓库做「信息聚合内容」本身就是一个创新。传统做法是建网站、做公众号、发小红书，但作者选择 GitHub——一个程序员聚集的平台——来分发 AI 副业内容。结果是被程序员 KOL 宝玉转发后爆火。这是渠道创新，不是内容创新。

🎯
公开透明 vs 付费韭菜

作者明确表态不做付费社群（觉得割韭菜），把所有内容开源。这在 AI 副业圈是比较少见的——大部分人选择收 199、299 卖课。仓库的 MIT 协议意味任何人都可以 fork、修改、重新发布。这种「先给价值再赚信任」的策略，实际上比直接收费更能积累长期资产。

📖
故事化包装：作者个人经历是内容的一部分

README 开头不是「欢迎来到 XXX 项目」的模板，而是一个真实的创业故事——从建站、爆火、衰落、被黑产劫持域名。这种「把作者经历嵌入产品叙述」的方式，在开源项目里很少见，但极有效果：读者看完故事已经建立了信任，后面的信息推荐就更愿意认真对待。

灵感卡片

看完这个项目我想到的产品点子——不一定用它的代码

灵感 1
「AI 副业地图」信息产品化

这个仓库本身就是一个 MVP 验证——17.5k Star 说明用户对「结构化 AI 赚钱路径」有强烈需求。可以做一个持续更新的导航产品，按「投入时间-预期收益-技能门槛」三维度给每条路径打分，让用户像选股票一样选副业。

灵感 2
「副业匹配测试」工具

大多数人对 AI 副业的困惑不是「有哪些」，而是「哪个适合我」。做一个 5 分钟测评工具：你每天有多少空闲时间？你会用哪些工具？你偏好创作型还是运营型？——然后推荐 3 条匹配路径 + 对应的入门教程。本质是「AI 副业版 MBTI」。

灵感 3
垂类 AI 工具的产品化机会

仓库里罗列的「AI 证件照」「AI 简历改写」「AI 老照片修复」都是已验证的刚需场景。但大部分还是教程级别的散装方案，没有形成产品。如果你能挑其中一个场景，做成「上传 → AI 处理 → 付费下载」的完整小程序，就是一个有付费意愿的生意。

灵感 4
「GitHub 作为内容平台」这个渠道值得研究

作者用 GitHub 分发非代码内容（教程、信息聚合）而且爆火，说明 GitHub 的受众（开发者）对高质量信息有付费意愿和传播力。如果你有面向开发者的内容产品，GitHub 是一个被低估的冷启动渠道。对比公众号/小红书，GitHub 的信任建立更快、传播链路更短。

灵感 5
Copycat × 本地化 = 可复用的产品模板

作者自己的路径就是 copycat → 本地化：把 typefully（海外 Twitter 写作工具）搬成 xaicreator（中文社媒写作工具）。这个模式可以套到很多场景：把海外验证过的 AI 工具搬运到国内市场，加上本地化（微信/小红书/抖音接口、中文 prompt 优化），就是一套低风险的产品打法。

产品化可行性

一个不会写代码的 PM，能不能用这个项目做出东西来？

零门槛
上手难度

不需要写任何代码。这是一个纯文档/信息聚合仓库，你只需要读就行。想把它变成产品则需要开发。

中等
依赖链

零依赖。没有任何安装步骤，不需要注册账号、不需要 API Key、不需要服务器。直接在浏览器里看。

非常适合
Vibe Coding 适配度

这个仓库做不到「靠 AI 对话直接运行」，因为它不是一个软件项目。但它天然适合作为 vibe coding 的素材源：挑一条变现路径，让 Claude Code 帮你搭 MVP。

如果你要做，大概需要什么

这个仓库本身不需要你做任何事——它就是个文档。但如果想从中孵化一个产品（比如挑一条 AI 副业路径做成工具），你需要：

• 时间：1-2 个月做出 MVP，选最窄的切入点（比如只做 AI 证件照，不做 AI 图片全家桶）

• 钱：初期基本可以零成本。Midjourney API / GPT API 的调用费用一个月几百块就够验证

• 人：PM 一个人 + Claude Code 写代码，完全能跑通。前提是你能用自然语言清晰描述产品逻辑

⚠️
注意

信息时效性风险：仓库的大多数内容来自 2023 年，AI 工具和平台政策已经发生很大变化（比如 Midjourney 封禁了大量中文关键词、抖音对 AI 生成内容的审核变严）。把它当灵感索引，不要当操作手册。

仓库已停止维护：作者已经转向其他项目，不会有更新。如果你 fork 了想继续维护，需要自己建信息更新机制。

版权隐患：仓库里的教程链接大部分是第三方内容（抖音教程、B站视频），作者是「聚合」而非「原创」。如果你要做类似的信息聚合产品，注意原作者权益。
