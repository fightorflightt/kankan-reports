#AI Agent #研究工具 #开源 #Skill

这是什么？

想象一下：你明天要见一个人，Google 只能给你他 2023 年的 LinkedIn。而这个工具能在几秒钟内搜遍 Reddit、X（Twitter）、YouTube、TikTok、Hacker News、Polymarket 预测市场——找到这个人最近 30 天在干什么、社区怎么讨论他、他发了什么代码、Podcast 说了什么。

它不是一个搜索引擎，它是一个AI agent 驱动的情报聚合器。用真实的点赞、评论、转发和真金白银的预测市场数据来排序，而不是靠 SEO 排名。

02

🎯 解决了什么痛点

为什么 46,000+ 人给这个项目点了 star？它回应了什么真实问题？

信息检索的体验，十年来没变过

你在 Google 搜一个话题。出来的前十条是 SEO 优化过的博客、三年前的新闻稿、加上一堆广告。真正有价值的讨论——Reddit 上那个 1,500 赞的真实用户评价、X 上行业专家昨晚发的长帖、YouTube 上一个 45 分钟的深度分析——Google 要么搜不到，要么埋在第十页。

如果你想了解一个人？LinkedIn 给你的是他 2023 年的职位。但真实情况是：他最近在 GitHub 上提交了 23 个 PR，在 X 上跟人争论 AI agent 的未来，在 Reddit 上被讨论了 569 个赞。

核心痛点：平台都是围墙花园

Reddit 有最真实的用户评价，但 Google 的爬虫碰不到评论区。X 有最快的突发信息和专家观点，但 ChatGPT 搜不了 X。YouTube 里有 45 分钟的深度分析，但 Gemini 虽然能搜 YouTube 却搜不了 Reddit。

每个平台都是一个围墙花园——它们不互通，各有各的 API，各有各的认证方式。如果你想全面了解一个话题，你得一个一个打开：Reddit → X → YouTube → Hacker News → 再搜一下 Polymarket 看看市场怎么押注的。这个过程要花 90 分钟。

04

💡 创新点

和常规搜索/研究工具比，它有什么不一样？

这不止是一个"聚合搜索"工具

表面上看，它就是把 Reddit、X、YouTube 搜了一遍然后汇总。但你仔细看它的设计，会发现几个很有意思的创新——这些创新才是它能冲到 46k star 的原因。

🧠
智能搜索：搜索之前先"理解"

普通搜索引擎是"关键词匹配"——你输入什么就搜什么。v3 版本在搜索之前先做一层"预研"：你搜"OpenClaw"，它先自动解析出这个产品的创始人 @steipete、相关 subreddit r/openclaw 和 r/ClaudeCode、正确的 YouTube 频道和 TikTok 标签——然后再去搜。这就像你问一个懂行的朋友，而不是问一个只会匹配字符串的机器人。

🔗
跨源故事合并

当同一个事件出现在 Reddit、X 和 YouTube 上时，v3 会把它们合并成一个"故事"而不是三条重复信息。比如说某音乐节在 Reddit 被讨论、在 X 上被转发、在 TikTok 上晒票价——v3 看到的是一条完整的故事线，而不是三个碎片。

😂
"最佳吐槽"评分

Reddit 和 X 上最精彩的内容往往是幽默的评论，而不是最"相关"的结果。v3 增加了一个"趣味性评分"维度——用第二个 AI judge 专门给每条结果打幽默分和病毒传播分。这样报告里不只是干巴巴的信息，还保留了社区最妙的神评论。这个设计在产品上很聪明：它让阅读报告这件事变得有趣。

📊
用真金白银排序

Polymarket 的赔率是有人拿真钱下注的结果。一个事件在 Polymarket 上有 6.6 万美元的交易量和 96% 的概率——这比任何评论员的猜测都有参考价值。把预测市场和社交媒体数据放在一起做综合排序，这在产品思路上很有意思。

📄
可分享的 HTML 简报

搜索完不只是聊天框里看看就没了。可以生成一个独立的 HTML 文件，深色模式、打印友好、离线可用。拖进 Slack 或邮件就能分享。这个功能很"产品"——它让研究结果从一个"聊天记录"变成了一个"可以传播的资产"。

🎯
按角色切换搜索方式

搜一个人的时候，搜索引擎不会自动切到"人模式"。但 v3 会：如果 topic 是一个人，它会自动从"谁提到了这个名字"切换成"这个人最近在干什么"——找他的 GitHub 提交记录、release notes、PR 合并率。你需要的是他的行为数据，不是提到他名字的新闻。

✨ 灵感卡片

看完这个项目我想到的产品点子——不一定用它的代码

灵感 1
「会前情报卡片」产品

想想你每次见客户、面试候选人、跟合作方开会之前，花多少时间搜索对方信息？能不能做成一个独立产品：输入对方公司名 / 姓名，自动生成一份一页纸的"会前简报"——最近动态、社区口碑、GitHub 活跃度（如果是技术人员）、社交媒体的真实讨论。面向销售、HR、投资人。

灵感 2
「产品决策雷达」

做产品的人经常要回答"我们该不该做 X？"这个项目告诉你：不用拍脑袋，去看社区在抱怨什么、竞品用户不满什么、哪些需求帖子点赞最多。做一个面向 PM 的工具：输入一个产品方向，自动聚合 Reddit、Hacker News、Product Hunt、X 上的用户真实声音，输出一份"要不要做"的信号报告。

灵感 3
「价格发现引擎」

这个项目用 Polymarket（预测市场）的数据来辅助判断。这个思路可以用到定价策略上：搜集社区对某类产品价格的讨论、竞品定价变化、用户愿意付多少钱的评论，结合市场价格趋势，给 PM 一个"建议定价区间"。定价一直是最难的产品决策之一，而社区里藏着大量付费意愿信号。

灵感 4
「行业脉搏 Daily Brief」

每天早上自动生成一份你的行业的"昨日要点"——不是新闻标题汇总，而是社区（Reddit/HN/X）真正在讨论什么、哪些帖子热度高、有什么新的吐槽或新工具被热议。像一份给产品人看的"早报"，但信号来源是社区真实讨论而非媒体通稿。用 HTML 简报格式，每天早上 Slack 推送。

灵感 5
「Vibe Coding 技能发现」

你正在探索怎么用 AI 做产品，而这个项目本身就是 Claude Code 的一个 skill。能否做一个"AI 工具 / skill 发现平台"——专门面向不会写代码的 PM，展示哪些 skill 能解决什么产品问题、上手难度如何、社区评价怎样。就像 Product Hunt 但聚焦在 AI agent skill 这个品类。

🔍 同类项目速览

它在整个生态里处于什么位置

市面上已有的方案，和它的位置

这个项目不是在跟 Google 竞争。它在做一个新的品类——AI agent 驱动的多源情报聚合。没有直接竞品能做到同样的覆盖面和评分方式。

Google 搜索 搜网页。碰不到 Reddit 评论区、X 帖子、Polymarket 赔率。SEO 驱动的排序。对"了解一个人最近在做什么"这类需求完全无力。
Perplexity AI 驱动的网页搜索，能搜 Reddit 但搜不了 X 和 TikTok。更像一个"带总结功能的搜索"，而非多源聚合。适合快速查事实，不适合"了解社区氛围"。
ChatGPT Search 和 Reddit 有合作，能搜到部分 Reddit 内容。但搜不了 X、TikTok、YouTube 评论区。且无法用 Polymarket 赔率或真实互动数据来排序。
手动调研 一个一个打开 Reddit → X → YouTube → HN。这是目前大多数人的做法，耗时 60-90 分钟。这个项目把它压缩到了几秒钟。
GummySearch Reddit 专用的用户调研工具，帮助产品人搜索 Reddit 上的用户反馈和讨论。但只限于 Reddit，没有多源聚合能力。属于"单源深度"路线。
📌
这个项目适合：

你需要快速了解一个话题、一个人、一个产品在"当下"的真实讨论和社区评价。特别是你要做决策、要见客户、要做产品判断之前。


不适合：

你需要做深度学术研究（去用 Perplexity 或 Google Scholar）。你只想查一个事实（直接 Google）。你需要长期趋势数据而非 30 天快照（这个项目聚焦最近 30 天）。
