这是什么？

一句话：这是一套「现成的 AI 专业团队」——把 232 个不同岗位的专家（前端、后端、UI 设计、增长、文案、测试、安全、财务、游戏开发……）提前写好了「人设 + 工作方法 + 交付标准」，你装进 Claude、Cursor 这类 AI 工具后，就能随时点名某个专家来帮你干活。

打个比方：平时你跟 AI 说「帮我写个页面」，它像个什么都会一点的临时工；而这套东西相当于给你配了一整层楼的专科同事，你喊一声「前端开发，来做个 React 组件」，它就切换成那个岗位的脾气、流程和专业标准来回应你。

应用场景

独立开发者搭 MVP：一个人想做小产品，但缺前端、后端、增长的同事——直接点名「前端开发 + 后端架构师 + 增长黑客」三个 agent，分工把雏形跑起来。
产品经理写方案 / 做调研：用「产品趋势研究员」「UX 研究员」「冲刺优先级排序」这些角色，帮你把模糊想法理成结构化的产品计划。
运营做内容营销：「内容创作者 + 小红书/Reddit 社区运营 + 数据分析师」组队，按平台各自的玩法产出一整套营销内容。
不想装、只想抄：把某个 agent 的「人设说明书」当模板，复制里面的思路和提示词，粘到自己平时用的 AI 对话里改改就用。

社区评价

起源于一个 Reddit 帖子，发布后头 12 小时就收到 50+ 条请求，随后滚成 11.9 万 star、353 次提交的现象级项目，全球贡献者持续补充新角色。（来源：项目 README / Acknowledgments）
第三方测评博客 YUV.AI 把它定位为「把你的 IDE 变成一个多 agent 的 AI 工作室」，强调它的卖点是「带人格的专科分工」而非又一个提示词清单。（来源：yuv.ai/blog/agency-agents）
社区自发出现了 8 种语言的本地化版本（中文、葡语、俄语、印尼语、阿拉伯语、韩语、日语），其中日语版甚至扩到 281 个本地化 agent——侧面说明它的「角色模板」结构被很多人认可、愿意二次创作。（来源：README / Community Translations）
02

解决了什么痛点

为什么有人愿意花几个月、拉上全球贡献者做这件事？它回应了什么真实问题？

用 AI 干活的人，大概都遇到过这些憋屈

痛点一：AI 是个「样样通、样样松」的临时工。你直接跟 AI 说「帮我写代码 / 写文案 / 做测试」，它给的东西往往很泛、很表面——因为它不知道你要的是哪个岗位的专业水准。一个资深前端会关心性能和像素级还原，一个随便应付的 AI 不会。每次你都得花大段话去「设定它是谁、该按什么标准做」，重复又费劲。

痛点二：提示词（prompt）攒了一堆，但都是散的、一次性的。很多人收藏过各种「神级提示词」，但它们是零散的纸条，今天用完明天就忘在哪了，也没有配套的工作流程和交付标准。缺的不是「一句好提示」，而是「一个能稳定按流程交活的专业角色」。（提示词 = 你发给 AI 的那段指令，决定它怎么回应你。）

痛点三：现成的 AI 工具是黑盒，改不动、也搬不走。市面上很多 AI 产品功能是封死的，你不能按自己的脾气和流程去调教它。而这个项目反过来——每个专家都是一份公开的、可以随便改、随便复制的「人设说明书」，你不满意就自己改，想搬到别的工具也行。

作者的出发点其实很朴素：与其每次都从零教 AI「你现在是谁、该怎么做事」，不如把这些「专业角色」一次性写扎实、攒成一个团队，谁要用谁拿走。

创新点

同样是「攒提示词」，凭什么它能冲到 11.9 万 star？

和常规做法不一样的地方

市面上「提示词合集」多如牛毛，但大多是一堆零散纸条。这个项目真正的创新不在「内容」，而在「把内容做成了产品」——它给每个角色配了人设、流程、交付标准，又用一套安装脚本把这些角色「装进」你正在用的各种 AI 工具里。换句话说，它把「收藏提示词」升级成了「雇佣一支随叫随到的专业团队」。

🎭
给 AI「立人设」，而不是只给一句指令

每个 agent 都有性格、口吻、做事原则和「成功标准」。比如测试角色会硬性要求「每条问题都得有视觉证据」。这种约束让 AI 的输出更像一个有职业操守的专家，而不是随便应付的万能助手。

📦
规模即壁垒：232 个角色 × 16 个分部

单个好提示词不稀奇，但系统化地覆盖工程、设计、营销、销售、财务、游戏、GIS……这么全的「岗位地图」，本身就是巨大的工作量和护城河。你几乎能为任何任务点到对口的人。

🔌
一套角色，处处能用（一键多工具分发）

配了转换+安装脚本，同一批角色能装进 Claude Code、Cursor、Copilot、Gemini 等十几个工具，还有原生 App 帮你一键安装、自动更新。你不用被某个工具绑死。（这正是上文聊过的 CLI 脚本 + 工具适配那套思路。）

🧩
能「组队」，不只是单兵

README 给了 6 个真实场景，演示怎么把多个角色拼成一支队伍协同干活（比如 8 个分部并行做一次完整产品探索）。它卖的不是单个工具人，而是「编队作战」的玩法。

🌍
开源 + MIT + 可 fork = 滚雪球的社区

05

灵感卡片

看完这个项目我想到的产品点子——不一定要用它的代码，但思路可以搬

灵感 1
给你的运营做一套「岗位人设」AI 助手

这个项目最值得抄的，是「把模糊的‘帮我写点东西’，拆成一个个有明确人设和交付标准的角色」。你完全可以为运营场景做几个固定角色：「群文案手」「商品卖点提炼师」「下单步骤图脚本师」。每个角色提前写死它的口吻、要素清单、输出格式，运营只管丢商品信息，AI 就按那个角色稳定产出——做单质量不再看个人手感。

灵感 2
「角色」比「功能」更好用，是个产品启发

普通用户面对一个空白 AI 输入框会发懵、不知道能问啥。这个项目证明：把能力包装成一个个有名字、有性格的「专家角色」，用户一下就懂「我该找谁」。这个思路能用在你们 App 的任何 AI 功能上——与其放个万能助手，不如给几张「专家卡片」让用户挑，转化和上手都会更好。

灵感 3
「成功指标」写进角色里，逼 AI 交高质量活

它每个角色都带「成功标准」（比如测试角色硬性要求挑出 3-5 个问题、要有证据）。这招很妙：与其事后挑 AI 的毛病，不如一开始就把验收标准塞进角色定义里。做单场景同理——给「文案手」角色定死「必须包含价格锚点 + 紧迫感 + 一句行动指令」，输出就不会飘。

灵感 4
「组队协作」模式：一个任务，多个角色接力

它演示了「8 个角色并行做一次产品探索」。对你做独立小工具特别有启发：一个稍复杂的任务（比如「调研一个选题」），可以让「趋势研究员→竞品分析→可行性核查」几个角色接力跑，每步专注一件事，最后拼成完整结论——这其实就是上文聊的「多 agent 协作」的平民玩法。

灵感 5
开源 + 可二次创作 = 让别人帮你扩内容

它靠「人人能改、能加角色」滚成了 232 个角色、8 种语言。放到运营上：如果你把「做单角色模板」做成一个开放、好改的格式，让一线代理自己沉淀和贡献新模板（哪个品类该怎么写），你的内容库就能靠团队自增长，而不是全压在中台一个人身上。

06

产品化可行性

一个不写代码的 PM，能不能用上这个项目？——能，而且门槛比想象低

低门槛
上手难度

有原生 App，下载后点几下就能把角色装进你的 AI 工具，全程不用碰命令行。这是它专门为「不想折腾」的人准备的入口。

低门槛
依赖链

前提是你已经在用 Claude Code / Cursor / Gemini 这类 AI 工具之一。本身免费、MIT 协议、可商用。不用注册、不用付费给它。

高适配
Vibe Coding 适配度

极高。哪怕不装，你也能直接把某个角色的「人设说明书」复制给我（Claude），让我照着那个角色帮你干活——零技术门槛。

如果你要用，大概需要什么

最轻的玩法（推荐你先试）：零成本、5 分钟。去仓库里挑一个跟你工作相关的角色（比如「内容创作者」「UX 研究员」），把那份说明书整段复制，粘给我，说「按这个角色帮我做 XX」。不装任何东西就能体验到「专家人设」的差别。

正式玩法：如果你想让这些角色常驻在自己电脑的 AI 工具里随时点用，就下载它的原生 App（macOS/Windows/Linux 都有），一键安装。需要你本机已经装了至少一个支持的 AI 工具。这步也基本是点鼠标，不写代码。

⚠️
注意几个坑

① 角色多≠都好用：232 个角色质量参差，有些是社区随手加的，别指望个个精品，挑你真正需要的几个就够。
② 它管「人设」，不管「干活的手」：这些角色本身只是说明书，真正干活的还是你背后的 AI 工具——效果好坏很大程度取决于你用的是不是好模型。
③ 偏英文语境：原始角色是英文写的、也偏欧美工作流（比如 Reddit、Twitter 营销）。直接套到国内私域/微信场景会水土不服，最好自己改写成中文、换成你的业务语境再用。
④ 装太多会拖累工具：README 自己也提醒，有的工具（如 OpenCode）装超过约 119 个角色会出问题，按需安装就好。

同类项目速览

「给 AI 配专家角色」这条赛道并不孤单，它在生态里处于什么位置

contains-studio/agents 这套 The Agency 的角色风格（前端魔法师、奇趣注入师、Reddit 社区运营等命名）与早期爆火的 contains-studio agents 一脉相承，可以理解为同一思路的「加强扩张版」——后者更精简，前者把规模堆到了 232 个。
wshobson/agents 另一个高人气的 Claude Code subagents 合集，风格更偏「工程严谨」、少人格戏份。适合纯技术团队；The Agency 则覆盖面更广（含营销、销售、财务、游戏）、更重人设。
VoltAgent / awesome-claude-code-subagents 同类的「subagent 大合集」，主打数量与分类齐全。和它们比，The Agency 的差异化在于「原生 App 一键多工具安装 + 自动更新」这套分发体验，对非技术用户更友好。
各类「Awesome Prompts」清单 最传统的对手——一长串 Markdown 提示词清单。差距在于：清单只给你「一句话」，The Agency 给的是「人设+流程+交付标准+安装方式」的完整角色，可用性高一个量级。
📌
这个项目适合：

已经在用 Claude Code / Cursor 这类 AI 工具、想要「按岗位点名专家」的个人和小团队；想找现成角色模板来抄思路的产品/运营；喜欢开源、想自己改写成中文业务版的人。


不适合：

完全不用 AI 编程工具、只想要个聊天机器人的人；需要中文私域/微信原生场景开箱即用的团队（得自己改造）；追求「少而精、每个都打磨过」的洁癖型用户（这里是大而全，质量参差）。
