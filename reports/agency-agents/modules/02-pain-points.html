解决了什么痛点

为什么 117,000 人给这个项目点了 Star？它回应了什么真实问题？

用 AI 干活的人，都在经历同样的困境

痛点一：「我每次都要重新写 prompt，而且写不好」。用 Claude 或 ChatGPT 干活时，大多数人的 prompt 就一句话：「帮我写个 React 组件」。AI 能写，但质量随机——有时候给的是过时的写法，有时候忽略了无障碍要求，有时候完全没考虑性能。而这个项目里的「Frontend Developer」Agent 有完整的身份设定、编码规范、测试策略、交付检查清单。你不需要懂怎么写 prompt，你只需要告诉它用哪个角色。

痛点二：「我换了工具，之前的 prompt 全废了」。今天用 Claude Code，明天团队换了 Cursor，后天老板说要用 Codex——每个工具的 agent 格式都不一样。The Agency 做了一件很聪明的事：同一套角色定义，通过一个 convert.sh 脚本自动转换成 18 种工具的格式。你只需要维护一套角色库，不用被工具锁定。

痛点三：「AI 像一个全能的实习生，但我要的是一个团队」。你需要的不是一个「什么都会一点」的通用 AI，而是一个前端工程师 + 一个安全审计员 + 一个 UI 设计师 + 一个增长黑客，各自在自己的领域是高手。232 个角色覆盖了软件开发从写代码到做营销的全链条——这本质上是把「一个人的 AI 助手」升级成了「一个 AI 公司」。
